2. 로지스틱 회귀 (Logistic Regression)
    1) 기본 분석 모델임
    2) 알고리즘 근간을 선형 회귀 분석에 두고 있음 
    3) 선형 회귀 분석은 연속된 변수를 예측하는 것임,
       이에 반해 로리스틱 회귀 분석은 Yes/No 처럼 두가지로 나뉘는 분류 문제를 다룸          

3. 문제 정의 
    1) 타이타닉호는 북대서양 횡단 여객선임
    2) 1912년 4월 10일 영국에서 미국 뉴욕으로 향하던 첫 항해 중 빙산과 충돌 침몰함 
    3) 타이타닉 데이터셋은 당시 승선한 승객의 정보임 
    4) 이름, 성별, 나이, 티켓 번호 등 정보가 실제 생존에 어떤 영향을 미치는지 확인하기 
        - 생존 여부를 예측하기        
    5) 문제 유형 - 분류 
    6) 평가 지표 - 정확도 
    7) 사용한 모델 - Logistic Regression        

4. 라이브러리 및 데이터 불러오기 
                                                                                티겟번호  승선한 항구   생존여부
5. 데이터 확인하기                                                               ------- ---------    --------
    1) Pclass	        Name	Sex	 Age  SibSp	                       Parch	Ticket	Embarked	Survived
       -------                           -------                       -----
       티켓 클래스                        함께 탑승한 형제 및 배우자 수    함께 탑승한 부모 및 자녀의 수
        
        - 독립 변수 8개와 종속변수 1개
        - Embarked (승선한 항구)
            - C(Cherbourg), Q(Queenstown), S(Southampton)
        - Survived (생존 유무)
            - 1(생존), 0(사망)            

    2) Non-Null Count
        - 빈 값(결측치)이 없음 

    3) Pclass (티켓 클래스)
        - 1, 2, 3 세가지 클래스가 있음 

    4) Age
        - 대부분이 35세 미만이지만, 최고 80세까지 있음      

    5) SibSp, Parch	                               
        - 대부분이 혼자 여행함 

    6) 상관관계
        - 각 변수의 상관관계 확인하기 
            - 두 변수 간의 변화가 서로 연관되었을 때, 
                - A가 증가할수록 B가 증가하거나 (양의 관계)
                    - SibSp(함께 탑승한 형제 및 배우자 수)와 Parch(함께 탑승한 부모 및 자녀의 수)의
                      상관관계가 가장 큼 
                - A가 증가할수록 B가 감소하거나 하는 관계를 숫자로 보여줌 (음의 관계)         
        - 상관관계의 강도 (절대값 기준)
            - 0.2 이하 : 상관관계가 거의 없음
            - 0.2 ~ 0.4
            - 0.4 ~ 0.6
            - 0.6 ~ 0.8 
            - 0.8 이상 : 매우 높은 상관관계 

6. 전처리 : 범주형 변수 변환하기 
    1) 자료형이 object인 변수들
        - 문자인 변수 
        - 기본적으로 ML 알로리즘에서는 문자열로 된 데이터를 이해 못함 
        - 숫자 데이터로 변환하는 기능 제공 

    2) object형을 숫자화하기 
        - 각 값(특정 문자)을 숫자로 대체하는 방법 
            - ex) 계절 
                - 봄, 여름, 가을, 겨울 = 1, 2, 3, 4   ==> 서열로 인식됨 

    3) 더미(dummy) 변수 활용하기 
        - ex) Sex 변수                                      

            Sex                              Sex-male           Sex-female
           -----        (one-hot)           --------------------------------    
           male         원-핫 인코딩             1                   0
           female    ------------------>        0                   1    
           male                                 1                   0
           male                                 1                   0

        - 고윳값의 개수를 확인  

7. 데이터 모델링 
    1) 독립변수, 종속변수, 훈련셋, 시험셋으로 나누기         

8. 예측 모델 평가하기 
    1) RMSE 
        - 실제값과 예측값의 차이를 각각 합산
    2) 시험셋 100개를 예측하고, 그 중 90개를 정확히 맞췄다면 
       정확도는 0.9가 되고, 모두 맞추면 1.0이 됨              


9. 피처 엔지니어링 (Feature Engineering)
    1) 기존 데이터를 손보아 더 나은 변수를 만드는 기법 
    2) 특징 공학, 특성 공학 
    3) 예) 더미 변수 만드는 일 

    4) 머신 러닝에 있어서 엄청 중요함 
        - 적합한 머신러닝 알로리즘을 선택하고 하이퍼파라미터를 튜닝하는 일 
        - 좋은 피처(ML)를 더 마련하는 것 
            - 피처라 함은 독립변수(통계학)의 다른 표현임 
    5) 피처 엔지니어링에서는 도메인 지식의 활용이 중요함
