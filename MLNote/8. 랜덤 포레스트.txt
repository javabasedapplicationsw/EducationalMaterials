1. 랜덤 포레스트 (Random Forest)
    1) 결정 트리의 집합체
        - 결정 트리를 사용하면 오버피팅 문제가 쉽게 발생함 
        - 여러 트리를 활용하여 최종 모델 만듦 
    2) 다양한 트리의 의견을 반영
        - 오버피팅 위험을 낮출 수 있음 
        - 여러 개의 트리를 만들 때는 데이터 전체를 사용하지 않음
        - 매번 다른 일부의 데이터를 사용하여 다른 트리를 만들어냄 
        - 일부 데이터만 사용하면 전체 데이터를 사용한 결과보다 예측력이 떨어짐 
        - 예측력이 다소 떨어지는 수많은 트리들이 모여 중윗값을 찾아냄  

    3) 결정 트리를 배깅 방식으로 결합한 모델 
        - 나무(tree)가 모여 숲(forest)을 이루는 형태 
        - 결정 트리가 모여 랜덤 포레스트를 구성됨 
        - 분류와 회귀 문제에 모두 적용 가능함 

    4) 랜덤 포레스트 구현     
        - RandomForestClassifier
        - RandomForestRegressor     

2. 하이퍼파라미터 
    1) 사용자가 직접 설정해야 하는 값
    2) 모델에도 최적 하이퍼파라미터를 전달해야 성능 좋음                       
    3) 모델이 좋은 성능을 내려면 하이퍼파라미터가 어떤 값을 가지면 좋을 지를 
       찾는 작업 => 하이퍼파라미터 최적화 
    4) 대표적인 최적화 방법
        - 그리드서치
        - 랜덤서치
        - 베이지안 최적화         

    5) 그리드서치 (grid search)
        - 가장 기본적인 하이퍼파라미터 최적화 기법 
        - 주어진 하이퍼파라미터를 모두 순회하며 가장 좋은 성능을 내는 값을 찾음 
        - 모든 경우의 수를 탐색하는 방식임
        - 하이퍼파라미터를 격자(grid)처럼 촘촘하게 순회하여 최적의 하이퍼파라미터 값을 찾는 기법임 
    
    6) 교차 검증(cross-validation)하여 성능을 측정하여 최종적으로 성능이 가장 좋았을 때의
       하이퍼파라미터 값을 찾아줌 

3. 탐색적 데이터 분석(EDA)
    1) 순서
        - 데이터 불러오기 --> 피처 엔지니어링 --> 데이터 시각화 --> 분석 정리   
    2) 모델링 전략 
        - 베이스라인 모델 
            - 데이터 불러오기 --> (기본적인)피처 엔지니어링 --> 평가지표 계산 함수 작성 --> 모델 훈련 --> 성능 검증 
        - 모델 성능 개선     
            - 피처 엔지니어링
            - 하이퍼파라미터 최적화 
            - 데이터 불러오기 --> 피처 엔지니어링 --> 평가지표 계산 함수 작성 --> 하이퍼파라미터 최적화 (모델 훈련) --> 성능 검증
