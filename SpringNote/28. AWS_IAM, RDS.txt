1. IAM
    1) Identity and Access Management 
    2) 클라우드 플랫폼에서 사용자의 신원과 접근 권한을 관리하기 위한 서비스 
    3) 사용자 권한은 로그인한 사용자 계정이 어느 서비스를 사용할 수 있는가?를 설정할 때 사용 
    4) 역할(role)은 AWS의 하나의 서비스가 다른 서비스에 접근할 수 있는가?를 설정할 때 사용

2. IAM 역할 생성 
    1) IAM 검색 후 액세스 관리 -> 역할 -> 역할 생성
    2) AWS  서비스 선택 후 EC2 선택 
    3) AWSElasticBeanstalk 검색 후 정책 3개 체크 
        - AWSElasticBeanstalkMulticontainerDocker
            - 오토스케일링, 블루/그린 배포시에 생성되는 EC2 서버는 
              Docker 이미지로 관리되기 때문에 컨테이너 관리에 필요한 권한 부여   
    4) 역할 이름 만들기            

3. RDS 생성 순서 
    1) RDS 검색 후 데이터 베이스 표준 생성 방식 
    2) MySQL 선택 -> 프리 티어 선택
    3) DB 이름 설정 (이름 상관없음)
    4) DB 관리자 이름 설정 (- cloudcoding)
    5) 사용자 암호 생성 (- cloudcoding0305)
    6) 퍼블릭 액세스 허용 
        - 로컬 컴퓨터에서 접속한 뒤 DB 생성, 테이블 생성해야 함 
    7) VPC 설정 
    8) 보안 그룹 새로 생성 (- 이름: rds-group)                        
    9) 자동 백업 활성화 중지
    10) 데이터 베이스 생성 버튼 클릭
        - 3~10분 소요 

    11) 엔드포인트 정보 확인         
    12) Workbench 설치 및 RDS 접속 
        - RDS에 로컬 컴퓨터 IP는 외부 접속이 허용되어 있기 때문에 원격 접속 가능 
        - 스키마 생성 (DB, Table)
