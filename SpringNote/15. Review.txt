1. Spring Framework
    1) Java 애플리케이션 매우 쉽고 빠르게 개발하기 위한 오픈 소스 Java 플랫폼 
    2) 2003년 아파치 2.0 라이선스로 6월 처음 릴리즈 됨 
    3) Java EE 플랫폼 위에 웹 애플리케이션 구축하기 위한 확장 기능이 핵심 기능임 
    4) POJO 기반 프로그래밍 모델을 베이스로 하여 J2EE 개발을 보다 쉽게 할수 있도록 지원 
    5) Module
        - 패키지와 클래스의 수가 많더라도 필요한 것만 추가하고 나머지는 무시하면 됨 
    6) 웹 MVC 
    7) 트랜잭션 관리 
        - 일관된 트랜잭션 관린 인터페이스 제공함 

2. 의존성 주입(Dependency Injection: DI)
    1) IoC(Inversion Of Control)의 DI임
        - IoC는 일반적인 개념이며 다양한 방식으로 표현될 수 있음 
        - DI는 IoC의 구체적인 방법 중 하나임 
    2) 종속성 주입
        - 두 클래스 간의 연결
    3) 응용 프로그램 객체를 서로 분리하는데 도움됨     

3. 관점 지향 프로그래밍 (Aspect Oriented Programming, AOP)
    1) 애플리케이션의 여러 지점에 결쳐 있는 교차 절단 관심사 (cross cutting concern)임 
    2) 응용 프로그램의 비즈니스 로직과 별개로 구분될 수 있음.
        - 로깅, 선언적 트랜잭션, 보안, 캐싱 등 
    3) OOP에서 핵심 단위는 클래스인 반면 AOP에서는 애스팩트(Aspect)임. 
    4) 영향을 받는 객체에서 교차 관심사를 분리하는 데 도움됨   

4. Configuration Metadata
    1) XML-based configuration
    2) Annotation-based configuration
        - 어노테이션 사용하여 의존성 주입을 구성함 
        - @Autowired, @Resouce, @Qualified...
    3) Java-based configuration
        - @Bean, @Configuration...        

5. @Component, @Controller, @Service, @Repository
    - 프로그램에 대한 데이터를 제공하는 메타데이터 형식임 
    - 프로그램에 대한 추가 정보 제공하는 데 사용됨 

6. @Component Annotation
    - @Component
        - @Controller
            : Presentation Layer
        - @Service
            : Service Layer
        - @Repository
            : Data Access Layer (DAO)

7. @Controller Annotation
    1) 특정 클래스가 컨트롤러 역할을 수행함 
    2) @RequestMapping 어노테이션을 기반으로 핸들러 메서드와 함께 사용됨 
    3) 클래스에만 적용할 수 있음
    4) 웹 요청 핸들러로 표시하는 데 사용됨 
    5) 주로 Spring MVC 애플리케이션과 함께 사용됨 
    6) 디스패처 서블릿은 메서드에 대한 @Controller 설정된 클래스를 스캔하고 
       @RequestMapping 감지함.

8. @Service Annotation
    1) 애플리케이션에서 비즈니스 로직은 서비스 계층 내에 상주함 
    2) @Service 어노테이션을 사용하여 클래스가 해당 계층에 속함을 나타냄            
    3) 클래스를 서비스 공급자로 표시하는 데 사용됨 
    4) 스프링 컨텍스트(AC)는 어노테이션 기반 구성 및 클래스 스캔이 사용될 때
       이러한 빈을 자동 감지 

9. @Repository Annotation
    1) 클래스가 객체에 대한 저장, 검색, 업데이트, 삭제 작업을 제공함 
    2) 빈 스캐팅을 통해 스프링 프레임워크에 의해 자동 감지됨 
    3) DAO 클래스가 데이터베이스 테이블에 대한 CRUD 작업 제공 담당 (DAO 패턴)
