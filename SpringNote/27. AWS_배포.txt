1. 배포 프로세스 
    1) 웹 애플리케이션 개발
    2) GitHub에 소스 코드 업로드 
    3) EC2에 프로젝트 다운로드 
        - EC2에 Git 프로그램이 설치되어 있음 
        - git clone 명령어 통해 해당 프로젝트 다운로드 
    4) EC2에 JDK 설치 
        - APT 명령어로 JDK 17 설치 
    5) 빌드
        - ./gradlew build
        - Gradle은 자바 기반의 오픈 소스 빌드 자동화 
    6) 실행 
        - 빌드를 하게 되면 실행파일인 JAR가 생성됨 
        - 해당 JAR 파일을 백그라운드로 실행 
        - nohup
            - no hang up                         
            - Unix, Unix-like OS에서 프로세스를 실행하고
              해당 프로세스가 터미널을 종료하거나, 
              세션이 끊겨도 종료되지 않고 계속 실행되도록 하는 명령어.
        - & 
            - 백그라운드에서 프로세스가 실행되는 명령어   

        - nohup java -jar aws2-1.0.jar &
        
        - nohup.out
            - java 실행 파일이 실행한 후 남는 로그 

    7) 모니터링 
        - tail 명령어로 어떤 요청이 들어왔는지 모니터링
