1. 내장 객체의 영역(Scope)
    0) 영역
        - 각 객체가 저장되는 메모리의 유효기간임 
    1) 내장 객체 중 4개는 영역이라는 개념을 가지고 있음 
        - page 영역
            - 동일한 페이지에서만 공유됨 
            - 페이지를 벗어나면 소멸됨 
        - request 영역
            - 하나의 요청에 의해 호출된 페이지와 포워드(요청 전달)된 페이지까지 공유됨
        - session 영역
            - 클라이언트가 처음 접속한 후 웹 브라우저를 닫을 때까지 공유됨  
        - application 영역 
            - 한 번 저장되면 웹 애플리케이션이 종료될 때까지 유지됨 

        - 웹에서는 페이지(page)들이 모여 하나의 요청(request)을 처리하며 
        - 요청들이 모여 하나의 세션(session)을 처리하며,
        - 다시 세션들이 모여 하나의 웹 애플리케이션(application)을 구성함 

2. JavaBeans   
    1) 데이터 전송 객체 (DTO, Data Transfer Object) 
        - 주로 데이터를 저장하거나 전송하는 데 쓰이는 객체 
    2) 다른 로직 없이 순수하게 데이터만 담고 있음 
    3) 데이터만 가지고 있는 객체 => 값 객체(Value Object, VO)
    4) DTO는 자바빈즈 규약에 따라 작성함     

    5) 자바빈즈 규약
        - 자바빈즈는 기본(default) 패키지 이외의 패키지에 속해야 함
        - 멤버 변수(속성)의 접근 지정자는 private으로 선언함 
        - 기본생성자가 있어야 함
        - 멤버 변수에 접근할 수 있는 게터(getter)/세터(setter) 메서드가 있어야 함
        - 게터/세터 메서드의 접근 지정자는 public으로 선언함 

3. page 영역        
    1) 기본적으로 클라이언트의 요청을 처리하는 데 관여하는 JSP 페이지마다 하나씩 생성됨
    2) 각 JSP 페이지는 page 영역을 사용하기 위해 pageContext 객체를 할당 받게 됨
    3) pageContext 객체에 저장된 정보는 해당 페이지에서만 사용할 수 있음
    4) 페이지를 벗어나면 소멸됨 
    5) include 지시어로 포함한 파일은 하나의 페이지로 통합되므로 page 영역이 공유됨  

4. request 영역
    1) 클라이언트가 요청을 할 때마다 새로운 request 객체가 생성됨
    2) 같은 요청을 처리하는 데 사용되는 모든 JSP 페이지가 공유함.
    3) request 영역에 저장된 정보는 현재 페이지와 포워드된 페이지까지 공유할수 있음.
    4) request 영역은 하나의 요청에 대한 응답이 완료될 때 소멸하게 됨    

5. session 영역
    1) 클라이언트가 웹 브라우저를 최초로 열고난 후 닫을 때까지 요청되는 모든 페이지는 
       session 객체를 공유할 수 있음
    2) 세션(session)
        - 클라이언트가 서버에 접속해 있는 상태 혹은 단위 
        - 회원인증 후 로그인 상태를 유지하는 처리에 사용됨 
        - 웹 브라우저를 닫으면 자동으로 로그아웃이 되는 이유가 바로 session 객체에 있음    

6. application 영역 
    1) 웹 애플리케이션은 단 하나의 application 객체만 생성 
    2) 클라이언트가 요청하는 모든 페이지가 application 객체를 공유하게 됨
    3) 웹 서버를 시작할 때 application 객체가 만들어지며,
       웹 서버를 내릴 때 삭제됨
       - application 영역에 한 번 저장된 정보는 페이지를 이동하거나,
         웹 브라우저를 닫았다가 새롭게 접속해도 삭제되지 않음

7. 요약
    1) page 영역 
        - 동일한 페이지에서만 공유됨, 페이지를 벗어나면 소멸됨
    2) request 영역
        - 하나의 요청에 의해 호출된 페이지와 포워드(요청 전달)된 페이지까지 공유됨
        - 새로운 페이지를 요청(페이지 이동)하면 소멸됨   
    3) session 영역
        - 클라이언트가 처음 접속한 후 웹 브라우저를 닫을 때까지 공유됨.
        - 포워드나 페이지 이동 시에도 영역은 소멸되지 않음
    4) application 영역
        - 한 번 저장되면 웹 애플리케이션이 종료될 때까지 유지됨.
        - 서버가 셧다운되지 않는다면 언제든지 공유되는 영역.
