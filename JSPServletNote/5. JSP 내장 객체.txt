1. 내장 객체(Implicit Object)
    1) 기본적인 요청과 응답, 화면 출력 등 거의 모든 웹 프로그래밍에 있어 필수임 
    2) 내장 객체는 JSP 페이지가 실행될 때 컨테이너가 자동으로 생성해줌 
    3) 별도의 객체 생성없이 각 내장 객체의 메서드를 즉시 사용할수 있음
    4) JSP 문서 안의 <% 스크립트릿 %>과 <%= 표현식 %>에서 사용 가능 

2. 내장 객체 종류 
    1) request
       response
       out
       session
       application
       pageContext
       config

3. request 객체
    1) 클라이언트(주로 웹 브라우저)가 전송한 요청 정보를 담고(저장하고) 있는 객체 
        - 클라이언트와 서버에 대한 정보 읽기 
        - 요청 매개변수에 대한 정보 읽기 
        - 요청 헤더 및 쿠키 정보 읽기       

4. response 객체 
    1) 요청에 대한 응답을 웹 브라우저로 보내주는 역할을 함 
    2) 페이지 이동을 위한 리다이렉트(redirect)
        - sendRedirect()
    3) HTTP 헤더에 응답 헤더 추가
        - 응답 헤더에 정보를 추가하는 기능 제공 

5. application 객체
    1) 웹 애플리케이션당 하나만 생성됨 
        - 웹 애플리케이션 전반에서 이용하는 정보 저장, 서버의 정보 등을 얻어오는 데 사용함 
    2) 모든 JSP 페이지에서 접근 할수 있음
    3) ServletContext 타입임.
