1. 서블릿 구현
    1) 클라이언트 요청을 전달할 '요청명'을 결정하는 일부터 시작함  
    2) 서블릿은 요청명을 기준으로 이를 처리할 서블릿을 선택하게 됨
    3) 요청명과 서블릿을 연결해주는 작업을 '매핑(mapping)'이라고 함
        - web.xml (배포 서술자, deployment description)
        - @WebServlet 애너테이션을 사용하여 코드에 직접 명시하는 방법 

2. web.xml에서 매핑 
                                     요청명
                                    --------
    1) http://localhost:8080/Chap01/hi/media
       --------------------- ------
       호스트(host)명        컨텍스트루트명  

    2) HiServiet 클래스는 HttpServlet 클래스를 상속함
        - doGet() 메서드를 오버라이딩함        

3. @WebServlet 애너테이션으로 매핑 
    1) web.xml에 비해 편리함
    2) 요청명(url-pattern)이 변경된다면 해당 서블릿 클래스를 수정한 후 다시 컴파일해야 함.

4. 자바 서블릿의 불편한 진실
    1) HTML을 작성?????
    2) HTML과 같은 화면(뷰)을 구성해 줄 템플릿 엔진이 필요함
        ==> JSP(JavaServer Pages)가 그 역할을 수행 
        - 템플릿 엔진 : 뷰를 구성할 템플릿에 문법 요소를 사용해 
                       데이터 출력들을 처리하기 위해 만들어진 라이브러리.

5. 서블릿의 수명주기(lifecycle) 메서드 
    0) 서블릿의 수명주기 관리는 서블릿 켄테이너가 담당함 
        - 각 단계마다 자동으로 특정 메서드를 호출함 (콜백 메서드)
    1) 서블릿 컨테이너는 클라이언트 요청이 들어오면 가장 먼저 서블릿 객체를 생성함 
        -> 서블릿의 초기화 -> 요청 처리함 
    2) 서버 종료를 종료할 때 서블릿 객체를 소멸시킴  

    3) 수명주기 메서드          전처리(@PostConstruct)
        - 서블릿 객체 생성 ---> init()
                                |
                               \|/
                            service()
                                |
                               \|/
                      doGet()        doPost()   
                                |
                               \|/
                            destroy()
                              후처리(@PreDestroy)
