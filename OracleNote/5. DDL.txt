1. 주요 데이터 타입(유형) 
    0) 테이블에 데이터가 저장되는 형태 
    1) CHAR(L)                                                     <-- 문자형
       CHARACTER(길이) 
        - 고정 길이 문자열
        - 고정 길이를 가지고 있으므로 L값보다 작을 경우 그 차이만큼 공백으로 채워 짐
        - 예) CHAR(5) : 컬럼에 4자리 문자열을 저장한 경우 => 맨끝에 1자리가 공백으로 채워짐 
    2) VARCHAR2(L)                                                  <-- 문자형
       CHARACTER VARYING(길이) 
        - 가변 길이 문자열 
        - L만큼의 최대 길이를 가짐
        - L값보다 작을 경우 해당 값만큼만 공간을 차지함 
        - 예) VARCHAR2(5) : 컬럼에 4자리 문자열을 저장한 경우 => 4자리 문자열만 저장함.
    3) NUMBER(L, D)                                                 <-- 숫자형
        - 정수, 실수를 저장함 
        - L값은 전체 자리수, D값은 소수점 자리 수   
    4) DATE                                                         <-- 날짜형
        - 날짜와 시각정보 
        - "년월일시분초"를 표현함 

    5) 데이터 유형은 데이터베이스의 테이블에 특정 자료를 입력할 때,
       그 자료를 받아들일 공간을 자료의 유형별로 나누는 기준      

    6) 선언한 유형이 아닌 다른 종류의 데이터가 들어오려고 하면 데이터베이스는 에러를 발생시킴   

2. CREATE TABLE 
    1) 테이블 명은 단수형 권고 
    2) 테이블 명은 같은 Owner내 다른 테이블과 중복되면 안됨
    3) 한 테이블내에서 컬럼 명이 중복되면 안됨 
    4) 테이블 생성 문 끝은 ';'로 끝나야 함 
    5) 데이터 유형은 반드시 지정 해야함 
    6) 테이블 명과 컬럼 명은 반드시 문자로 시작해야 함 
    7) A-Z, a-z, 0-9, _, $, # 문자만 허용    

3. CTAS 
    - CREATE TABLE AS SELECT
    - SELECT문을 기반으로 CREATE TABLE을 할수 있는 CREATE문
    - 문법
        - CREATE TABLE NEW_TABLE
          AS 
          SELECT문   
          ;    

        - CREATE TABLE NEW_TABLE (NEW_COLUMN_1, NEW_COLUMN_2)
          AS 
          SELECT문  
          ;  
        
        - CREATE TABLE IF NOT EXISTS NEW_TABLE
          AS 
          SELECT문    
          ;      



4. 제약조건 (CONSTRAINT)
    1) 제약조건이란 사용자가 원하는 조건의 데이터만 유지하기 생성함 
    2) 데이터의 무결성을 유지하기 위한 데이터베이스의 보편적인 방법으로 
       테이블의 특정 컬럼에 설정하는 제약 
    
    3) 제약 조건의 종류
        - 기본 키 (Primary Key)
            - 테이블에 저장된 행을 고유하게 식별하기 위함
            - 하나의 테이블에 단 하나의 기본 키만 정의 가능 
            - 기본 키 생성 시 DBMS는 자동으로 UNIQUE 인덱스를 생성
            - 기본 키 컬럼에는 NULL 입력 불가 
        - 외래 키 (Foreign Key) 
            - 다른 테이블의 기본 키를 외래 키로 지정하는 경우 생성함 
            - 참조무결성 제약조건  
        - 고유 키 (Unique Key)
            - 테이블에 저장된 행 데이터를 고유하게 식별하기 위해 생성
            - NULL 입력 가능   
        - NOT NULL 
            - NULL 값의 입력을 금지 
            - 필수적으로 값이 들어가야하는 컬럼이 됨 
        - CHECK
            - 입력할 수 있는 값 종류 및 범위를 제한 함                                             

5. NULL 
    1) 공백이나 숫자 0과는 전혀 다른 값임
    2) 공 집합과도 다름 
    3) "NULL"
        - 아직 정의되지 않은 미지의 값
        - 현재 데이터를 입력하지 못하는 경우 
            - 데이터 입력 시에 컬럼의 값이 지정되어 있을 않을 경우 기본값(DEFAULT)을 사전 설정할수 있음
            - 데이터 입력 시 명시된 값을 지정하지 않은 경우에 NULL 값이 입력됨
