1. 데이터 모델링
    1) 테이블(데이터)의 근간이 되는 것이 데이터 모델
    2) 데이터 모델을 설계하는 과정이 데이터 모델링임.

2. 모델링
    1) 우리가 살고 있는 3차원의 현실 세계를 단순화하여 표현하는 것 (단순화)
    2) 현실 세계를 추상화하여 그 구조를 표현한 것 (추상화)
    3) 모델링이란 현실 세계에 존재하는 사물이나 사건에 관한 관점 및 
       양상을 연관된 주체(사람 혹은 그룹)를 위하여 명확하게 하는 것 (명확화)

3. 모델링의 최종 정의
    1) 복잡한 현실 세계를 추상화, 단순화, 명확화하기 위해 
       일정한 표기법으로 모델을 표현하는 기법    

4. 데이터 모델링
    1) 현실 세계의 비즈니스를 IT 시스템으로 구현하기 위해 데이터 관점으로 업무를 분석하는 기법 
    2) 현실 세계의 비즈니스를 약속된 표기법으로 표현하는 과정 
    3) IT 시스템의 근간이 되는 데이터베이스를 구축하기 위한 분석 및 설계의 과정 
    4) 대부분의 IT 시스템은 관계형 데이터베이스(RDBMS) 기반으로 구축되어 있음  
    5) 데이터 모델링을 통해 정의된 데이터 모델을 기반으로 
       물리적인 데이터베이스가 구축되고
       SQL문을 활용하여 데이터가 입력, 수정, 삭제, 조회되고 있음.

       데이터 모델링     ==>   데이터 모델   ==> 물리 DB          <====== 사용자 
                                                                INSERT
                                                                UPDATE
                                                                DELETE
                                                                SELECT                  

5. 데이터 모델링 3단계 
    1) 현실세계  -------------->     개념적 구조 
                개념 데이터 모델링        |
                                         |  논리 데이터 모델링 (기본키, 속성, 외래키 등)
                                        \|/   
      물리 구조  <-------------      논리적 구조 
        (DB)    물리 데이터 모델링
                 (구체적)

6. 데이터 모델링 3요소
                                              집합                                         단수
                                              ------------------------------------------------------------------------------
    1) 업무가 관여하는 어떤 것(Things)   ->     엔터티(Entity)   ex) 사원               ex) 이순신 사원                               
                     -----------------    
    2) 어떤 것이 가지는 성격(Attributes)    -> ex) 사원은 사원명, 생년월일을 가짐        ex) 사원명 이순신은 생년월일은 200101이다 
                       ----------------
    3) 업무가 관여하는 어떤 것 간의 관계(Relationships)   -> ex) 사원은 부서에 속함      ex) 이순신 사원은 개발팀에 속함                    
                                  -------------------

7. ERD (Entity Relation Diagram) 그리는 순서
    1) 엔터티를 그리고 속성을 정의함 
    2) 1번에서 그린 엔터티를 적절하게 배치함 
    3) 각 엔터티 간의 관계를 설정 
    4) 설정한 관계의 관계명을 기술함
    5) 설정한 관계의 참여도를 기술함 
    6) 설정한 관계의 필수 여부를 기술 

    7) Case Method 표기법 사용
        - Barker(바커) 표기법이라고도 부름 

8. 엔터티(Entity) 
    1) 비지니스 관점에서 IT 시스템을 통해 
       저장 및 관리해야 할 집합적인 어떤 것(Thing).
    2) 엔터티는 사람, 사물, 사건, 개념 등의 명사에 해당
    3) 엔터티는 비즈니스 관점에서 IT 시스템을 통해 관리가 필요한 관심사에 해당

    4) 엔터티             인스턴스 
       ------------------------------
       상가              스타벅스 강동점   
                         이디야 천호점 
       지하철역           5호선 천호역 
                         8호선 강동역                   
   5) 하나의 엔터티는 여러 개의 인스턴스를 가짐 
      엔터티는 인스턴스의 집합임      

9. 엔터티의 표기법 
    1) 모서리가 둥근 사각형 (바커 표기법 기준)을 그린 후 맨 위에 엔터티명을 기재함 

10. 엔터티 특징
    1) 요구 조건을 만족을 위해 반드시 필요하고 저장 및 관리하고자 하는 정보여야 함 
        - 예) 상가, 지하철역, 사원,...
    2) 유일한 식별자에 의해 식별이 가능해야 함.
       집합 내에서 단 1건을 콕 짚어낼 수 있어야 함
        - 예) 상가번호, 지하철역번호, 사원번호 
    3) 영속적으로 존재하는 인스턴스의 집합(2개 이상)이어야 함
        - 예) 상가는 여러 개이다. 지하철역은 여러 개이다.
    4) 엔터티는 비즈니스 프로세스에 의해 반드시 이용되어야 함 
    5) 엔터티는 반드시 속성을 가지고 있어야 함 
        - 예) 상가 -> 상가명, 지하철역 -> 역명 
    6) 엔터티는 다른 엔터티와 최소 1개 이상의 관계가 있어야 함                 

11. 엔터티 분류 
    1) 유무형에 따른 분류
        - 유형 
            - 실체가 존재하고 물리적인 형태가 있음
            - 예) 고객
        - 개념 
            - 물리적인 형태가 존재하지 않음
            - 비지니스적으로 관리해야 할 개념적 정보를 저장하는 엔터티 
            - 예) 보험상품
        - 사건 
            - 비즈니스를 수행함으로써 발생되는 엔터티 
            - 예) 계약 
    2) 발생시점에 따른 분류 
        - 기본 (Key, 키)
            - 독립적으로 생성이 가능한 엔터티
            - 기본 엔터티는 중심엔터티의 부모 역할을 하게 됨
            - 예) 고객, 상품 
        - 중심 (Main, 메인)
            - 기본 엔터티로 부터 발생되며, 비즈니즈에 있어서 중심적인 역할을 하는 엔터티 
            - 예) 주문
        - 행위 (Action, 액션)
            - 2개 이상의 부모 엔터티로부터 발생되는 엔터티 
            - 예) 주문목록 

12. 엔터티의 명명 
    1) 가급적 업무 담당자들(현업)이 사용하는 용어를 사용함
    2) 엔터티는 단수명사여야 함 
    3) 엔터티의 이름은 해당 모델 내에서 유일한 이름이어야 함
    4) 엔터티의 생성 의미에 맞게 이름을 부여함
