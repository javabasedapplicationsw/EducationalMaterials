1. 개념 
    1) join이 필요한 이유 
        - 필요한 정보(상품명, 유저 정보, 카테고리 정보 등)가 
          각기 다른 테이블에 분산 저장되어 있을 때,
        - 이를 하나의 테이블로 가져와 보기좋게 데이터를 추출하기 위해서임.
    2) 서로 다른 테이블을 공통 컬럼을 기준으로 합치는(겹합하는) 테이블 단위 연산.     
        - 여러 테이블에 대한 조인 검색 
            - 두 개 이상의 테이블들을 연결 또는 결합하여 데이터를 출력하는 것
        - 여러 개의 테이블을 연결하여 데이터를 검색하는 것 
        - 조인 속성
            - 조인 검색을 위해 테이블을 연결해주는 속성(컬럼)
            - 일반적인 경우 PK(PRIMARY KEY)와 FK(FOREIGN KEY)의 값 연관에 의해 
              조인이 이루어짐.
            - PK, FK 관계와는 별도로 일반 컬럼끼리 조인이 이루어지는 경우도 있음  

    3) 문법 
        -   SELECT * 
            FROM 테이블1, 테이블2 
            WHERE 테이블1.컬럼명 = 테이블2.컬럼명;     

        - (ANSI 표준)
            SELECT * 
            FROM 테이블1 
            (INNER) JOIN 테이블2
            ON 테이블1.컬럼명 = 테이블2.컬럼명;     

        ------------------------------------------------
        -   SELECT *
            FROM 테이블1, 테이블2, 테이블3
            WHERE 테이블1.컬럼명 = 테이블2.컬럼명
            AND 테이블2.컬럼명 = 테이블3.컬럼명

        - (ANSI 표준)     
            SELECT *
            FROM  테이블1      
            (INNER) JOIN 테이블2
            ON 테이블1.컬럼명 = 테이블2.컬럼명
            (INNER) JOIN 테이블3
            ON 테이블2.컬럼명 = 테이블3.컬럼명

    4) 최소로 필요한 조인 조건의 개수 = 조인되는 테이블 수 - 1 
        - 4(조인되는 테이블의 수) - 1 = 3 (최소로 필요한 조인 조건의 개수)
        - 3(조인되는 테이블의 수) - 1 = 2 (최소로 필요한 조인 조건의 개수)
        - 2(조인되는 테이블의 수) - 1 = 1 (최소로 필요한 조인 조건의 개수)           

2. 조인의 형태 
    1) INNER JOIN 
        - 내부 조인 
        - JOIN 조건에서 동일한 값이 있는 행만 반환 
        - 조인 시 서로 다른 테이블에 같은 컬럼명이 존재하면  
          구분을 위해 테이블명.컬럼명으로 사용해서 표시함.
        - 연결하려는 테이블 간에 조인 속성(컬럼)명은 달라도 되지만 도메인은 같아야 함 
        - 일반적으로 외래키(FK)를 조인 속성으로 이용함   
        - 조인 시 NULL값을 허용하지 않음  

    2) NATURAL JOIN 
        - 두 테이블 간의 동일한 이름을 갖는 모든 컬럼들에 대해 EQUI(=) JOIN을 수행 
        - 두 테이블이 공통적으로 가지고 있는 컬럼으로 자동으로 조인됨 
        - 조인 컬럼은 앨리어스를 주면 안됨 

    3) USING 조건절 
        - NATURAL JOIN에서는 모든 일치되는 컬럼들에 대해 JOIN이 이루어지지만,
          FROM 절의 USING 조건절을 이용하면 같은 이름을 가진 컬럼들 중에서
          원하는 컬럼에 대해서만 선택적으로 EQUI(=) JOIN을 수행 
        - USING 절에 들어가는 조인 컬럼에 앨리어스를 쓸 수 없음 

    4) ON 조건절
        - JOIN 서술부(ON 조건절)와 비 JOIN 서술부(WHERE 조건절)
        - 컬럼 명이 다르더라도 JOIN 조건을 사용할수 있음 
        - ON절에 조인 컬럼을 기재함 
        - ON절 내에 조인 컬럼에 앨리어스를 사용해야 함 
        - 앨리어스를 정확히 기재하지 않으면 에러가 발생함 

    5) OUTER JOIN         
        - 두 집합에서 어느 한쪽 집합은 모두 출력하고 
          어느 한쪽 집합은 매칭 되는 것만 출력하는 조인 연산 방식 
        - JOIN 조건에서 동일한 값이 없는 행도 반환할 때 사용할 수 있음
        - 종류 
            - LEFT OUTER JOIN 
                - 두 집합에서 LEFT 집합은 모두 출력하고 
                  RIGHT 집합은 매칭 되는 것만 출력하는 조인 연산 방식                
            - RIGHT OUTER JOIN 
            - FULL OUTER JOIN 
                - 왼쪽 테이블 집합, 오른쪽 테이블 집합을 모두 리턴함
                - LEFT OUTER JOIN +  RIGHT OUTER JOIN + INNER JOIN의 결과 집합 모두가 출력됨
